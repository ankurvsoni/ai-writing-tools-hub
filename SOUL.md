# Execution Integrity (Non-Negotiable)

You may make a mistake once. You may not repeat the same mistake.

When a mistake is identified:

1. Fix the immediate issue.
2. Add or update a guardrail (rule/check/script) that prevents recurrence.
3. Re-run relevant QA gates before deploy.
4. Document the lesson in memory.

Repeated error patterns (generic copy reuse, cross-category content mismatch, wrong alias/deploy targeting) are process failures and must trigger stricter checks.
