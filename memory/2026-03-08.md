# 2026-03-08

## Session learnings (affiliate-content-hub)

- Confirm project target first by checking:
  - active git branch
  - Vercel linked project (`.vercel/project.json` / `vercel link`)
  - custom-domain alias map (`vercel alias ls`)
- A successful deploy to the wrong project/domain is still a failure for the user.
- `reviews.orkolabs.com` can lag behind if still aliased to an older deployment; fix with:
  - `vercel alias set <latest-deployment> reviews.orkolabs.com`
- Styling bug root cause:
  - page wrapper used `.content`, but Amazon link CSS targeted `.container`.
  - Fix by targeting both `.content` and `.container`.
- Duplicate cart icons came from two sources:
  - inline icon in CTA text + CSS `::before` icon.
  - Keep one source only.
- Duplicate header/disclosure UX issues came from template + markdown overlap.
  - Avoid rendering repeated H1/disclosure blocks in page template when content already includes them.
- For affiliate pages, generic lines are immediately visible to users; prefer concrete buyer-language:
  - what it is
  - best for (specific persona)
  - what stands out (specific differentiator)
  - before-you-buy checks
  - skip-if criteria
- Operational guardrail: after any fix, verify both:
  - direct Vercel deployment URL
  - production/custom domain URL

## Process improvements to keep

- Before claiming "deployed", run this checklist:
  1. `git rev-parse --abbrev-ref HEAD`
  2. `vercel whoami`
  3. `vercel alias ls --scope ankurvsonis-projects | grep <domain>`
  4. open/fetch both deployment URL and live domain URL for the same page
- When user reports mismatch with screenshot, assume domain alias mismatch first, then cache.
